Continuous Delivery / Livraison continue

Qu'est-ce que la CD (Continuous Delivery) ?

La CD (Continuous Delivery) est une pratique qui permet de préparer le logiciel pour la livraison.

Exemple : Créer une image docker et la pousser sur un registry.

Dockerfile

Un Dockerfile est un fichier qui contient des instructions pour construire une image docker.

Exemple de Dockerfile

					
FROM node:16-alpine
WORKDIR /app
COPY . /app
RUN npm install
CMD ["npm", "start"]

Créer une image Docker à partir d'un Dockerfile

					
docker build -t my-image .

Pousser une image Docker sur un registry

					
docker login
docker tag my-image username/my-image:tag
docker push username/my-image:tag

Exemple de pipeline CD

					
pipeline {
	agent {
		label 'agent name'
	}

	environment {
		REGISTRY = 'username'
		TAG = 'tag'
		DOCKER_HUB_CREDENTIALS = credentials('docker-hub-credentials')
	}

	stages {
		//...
		stage('Continuous Delivery') {
			steps {
				sh 'docker build -t my-image .'
				sh 'docker tag my-image $REGISTRY/my-image:$TAG'
				sh 'docker login -u $REGISTRY -p $DOCKER_HUB_CREDENTIALS'
				sh 'docker push $REGISTRY/my-image:$TAG'
			}
		}
		//...
	}
}
